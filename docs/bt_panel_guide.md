# 宝塔面板 (BT Panel) 部署指南 (针对第 5 步及之后)

如果您使用宝塔面板，可以极大简化 SSL 证书申请、Nginx 反向代理以及 Node.js 进程管理的操作。以下是接续 `deployment_guide.md` 第四步后的宝塔专属操作指引。

## 1. Node.js 项目管理 (接续第 4.3 节)

根据您的报错 `Error: Cannot find module 'uuid'`，这是因为两个原因：
1.  `package.json` 中缺失了 `uuid` 依赖声明。
2.  服务器上没有执行 `npm install` 安装依赖包。

**解决办法**:

1.  **更新 package.json**: 我已经更新了本地的 [package.json](file:///e:/Project/QtProject/ChatApp/server/package.json)，添加了 `uuid`。请重新上传该文件到服务器。
2.  **安装依赖**:
    *   在添加项目弹窗中，**不要勾选** “不安装node_module” (不安装node_module)。勾选它会导致宝塔跳过安装过程，从而找不到模块。
    *   如果项目已经添加，在宝塔“Node项目”列表点击 **“项目名称”** -> **“模块”**，点击 **“一键安装”** 按钮。
3.  **刷新并启动**:
    *   此时在 **“启动选项”** 出会看到 **`start`**，选择它并确定。
    *   **Node版本**: 16.9.0 是支持的。因为我已将 `uuid` 降级到兼容版本 `9.0.1`。您可以放心使用 16.9.0。

## 2. 设置网站入口与 SSL (接续第 5 步)

在宝塔中，**“添加站点”** 是管理域名、SSL 证书和 Nginx 配置的入口。虽然我们的后端是 Node.js，但仍需创建一个“站点”来作为对外开启 HTTPS/WSS 访问的门面。

### 2.1 创建入口站点
1.  点击左侧 **“网站”** -> **“添加站点”**。
2.  **域名**: 输入 `59.110.44.145`。
3.  **PHP版本**: 选择 **“纯静态”** (注意：这里选纯静态即可，因为我们不需要 PHP，稍后会用反向代理接管流量)。
4.  **网站根目录**: 随便选一个空的目录即可（它不会被真正用到，因为请求会被转发给 Node.exe）。

### 2.2 申请 SSL 证书 (HTTPS/WSS 必备)
1.  在网站列表中找到 `59.110.44.145`，点击右侧的 **“设置”**。
2.  在弹窗中选择左侧菜单的 **“SSL”**。
3.  选择 **“Let's Encrypt”**。
4.  勾选您的域名，点击 **“申请”**。

> [!WARNING]
> **出现 403 错误的排查方案**:
> 如果您遇到类似 `Invalid response ... 403` 的验证失败，请按以下顺序排查：
> 1.  **检查 WAF/防火墙**: 在宝塔左侧菜单点击 **“WAF”** (如果有安装)，暂时关闭它。WAF 经常会拦截 `.well-known` 目录的访问。
> 2.  **检查目录权限**: 进入 **“文件”**，找到您网站的根目录，确保权限是 `755`，所有者是 `www`。
> 3.  **冲突排查**: 如果您已经在第 1 步中开启了 Node.js 项目并绑定了 `59.110.44.145` 域名，请先在“Node项目”里**解绑域名**，等 SSL 申请成功后再绑回来。因为 Node 项目会抢占 Nginx 对该域名的控制权，导致 Let's Encrypt 找不到验证文件。
> 4.  **手动测试**: 在根目录下手动创建文件夹 `.well-known/acme-challenge/`，放一个测试文件，看浏览器能否访问 `http://59.110.44.145/.well-known/acme-challenge/测试文件`。
> 5.  **备选方案**: 如果 HTTP 验证始终失败，请改用 **“DNS 验证”**。这需要您去阿里云 DNS 管理界面添加一条 `TXT` 记录，这是 100% 成功的方案。

5.  申请成功后，开启 **“强制 HTTPS”** 开关。

## 3. 配置反向代理 (支持 WSS)

这是最关键的一步，它将 HTTPS 连接转发到您的 Node.js 端口，并支持 WebSocket。

1.  仍在网站设置弹窗中，点击左侧菜单的 **“反向代理”**。
2.  点击 **“添加反向代理”**。
3.  **代理名称**: `ws_proxy`。
4.  **目标 URL**: `http://127.0.0.1:8080`。
5.  **发送域名**: `$host`。
6.  点击 **“提交”**。
7.  **开启 WebSocket 支持**:
    *   点击刚创建的代理右侧的 **“配置文件”**。
    *   确保配置中包含以下三行（如果不包含，请手动加入）：
        ```nginx
        proxy_set_header Upgrade $http_upgrade;
        proxy_set_header Connection "upgrade";
        proxy_set_header Host $host;
        ```
    *   点击 **“保存”**。

## 4. 客户端打包与部署 (第 6 步)

宝塔端配置完成后，您的生产环境地址正式确认为：`ws://59.110.44.145:8080`。

### 4.1 确认驱动
由于您是直连数据库，请务必在本地打包时，从 PostgreSQL 安装目录的 `bin` 下拷贝以下文件到打包目录：
*   `libpq.dll`
*   `libintl-x.dll` (x 为版本号，如 8 或 9)
*   `libiconv-x.dll`
*   `libcrypto-x.dll`
*   `libssl-x.dll`

### 4.2 运行与测试
1.  启动打包后的 `ChatApp.exe`。
2.  测试登录提示。
3.  如果无法连接，请检查宝塔面板 **“安全”** 菜单（防火墙）是否放行了 `80`, `443`, `8080` 端口。

---

> [!TIP]
> **常见问题排查**:
> *   **SSL 申请失败**: 请检查域名是否已正确解析到该服务器 IP，且 80 端口没有被占用。
> *   **WebSocket 断连**: 检查 Nginx 反向代理配置中的 `proxy_read_timeout`，如果聊天容易断开，可以将其设置为 `86400` (24小时)。
