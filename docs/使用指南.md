# ESChat 使用指南

本文档面向日常开发和联调使用，重点覆盖当前仓库的实际启动方式和常见界面操作。

## 启动顺序

### 后端

推荐直接运行：

```powershell
./scripts/start-local-server.ps1
```

或者：

```bash
cd server
npm start
```

### 客户端

本地联调不要直接双击发布版 exe，优先运行：

```powershell
./scripts/start-local-client.ps1
```

这个脚本会把客户端固定到本机 127.0.0.1:8080，避免落到 Release 默认远端地址。

## 登录与注册

### 注册

1. 打开登录页。
2. 点击“立即注册”。
3. 输入用户名和密码。
4. 注册成功后返回登录页。

### 登录

1. 输入用户名。
2. 输入密码。
3. 如有需要勾选“记住我的登录信息”。
4. 点击“登录”。

登录成功后，窗口会切换到主界面，并开始加载会话、联系人和个人资料。

## 主界面结构

### 侧边栏

- 消息页：查看会话列表与聊天区
- 联系人页：查看好友和联系人详情
- 通知页：查看好友申请和系统通知
- 设置页：查看设置和账号操作

### 顶部用户信息入口

点击标题栏中的当前用户名区域，可打开个人资料弹窗。

## 常用操作

### 联系人页

1. 点击左侧联系人图标。
2. 在搜索框中筛选联系人。
3. 点击联系人后，可从联系人详情进入聊天。

### 聊天页

1. 点击左侧消息图标。
2. 在会话列表中选择一个会话。
3. 在底部输入框中输入消息。
4. 按 Enter 发送。

### 资料页

1. 点击标题栏中的用户名区域。
2. 在弹窗中查看头像、昵称、地区等资料。
3. 点击“编辑资料”后再修改并保存。

## 联调建议

### 本机 smoke check

建议至少点一遍：

1. 联系人页
2. 聊天页
3. 个人资料弹窗

### 服务检查

启动后可验证：

- [首页](http://127.0.0.1:8080/)
- [会话接口](http://127.0.0.1:8080/api/conversations)

会话接口在未带认证时返回未授权是正常现象，只要服务有响应即可。

## 常见问题

### 客户端连错地址

现象：本地后端正常，但客户端仍然请求远端。

处理：使用 scripts/start-local-client.ps1 启动，而不是直接运行 bin/ChatApp.exe。

### 数据库连接失败

优先检查：

1. PostgreSQL 是否启动。
2. server/.env 是否正确。
3. chatapp 数据库是否已经执行过 server/init_db.sql。

### 页面打开但数据为空

优先确认当前账号是否已有好友、会话和资料数据；很多页面在空账号下仍可正常打开，但列表会为空。
