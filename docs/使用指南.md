## 启动后端服务

### 1. 检查 PostgreSQL 数据库

**Windows:**
```bash
# 检查 PostgreSQL 服务状态
sc query postgresql-x64-14

# 如果未启动，启动服务
net start postgresql-x64-14
```

**Linux/macOS:**
```bash
# 检查服务状态
sudo systemctl status postgresql

# 启动服务
sudo systemctl start postgresql
```

### 2. 初始化数据库（首次运行）

```bash
# 进入数据库目录
cd E:\Project\QtProject\ChatApp\database

# 使用 psql 执行初始化脚本
psql -U postgres -f init.sql
```

或使用 Qt 客户端自动初始化（已在代码中实现）。

### 3. 配置后端环境变量

编辑 `server/.env` 文件：

```env
# 服务器端口
PORT=8080

# PostgreSQL 数据库配置
DB_HOST=localhost
DB_PORT=5432
DB_NAME=chatapp
DB_USER=postgres
DB_PASSWORD=123456

# JWT Token 密钥（生产环境请修改）
JWT_SECRET=eschat_secret_key_2024
```

### 4. 安装后端依赖

```bash
cd E:\Project\QtProject\ChatApp\server
npm install
```

### 5. 启动 Node.js 服务器

**方式一：直接启动**
```bash
cd E:\Project\QtProject\ChatApp\server
node index.js
```

**方式二：使用 npm 脚本**
```bash
cd E:\Project\QtProject\ChatApp\server
npm start
```

**成功启动输出示例:**
```
✅ PostgreSQL connected: 2026-03-04 10:30:15.123+00
Server running on http://localhost:8080
✅ Database: chatapp@localhost:5432
```

**如果数据库连接失败:**
```
❌ Database connection failed: connect ECONNREFUSED 127.0.0.1:5432
Please check:
  1. PostgreSQL service is running
  2. Database credentials in .env file
  3. Database exists or user has permission to create it
```

### 6. 服务进程信息

| 进程名 | 端口 | 说明 |
|--------|------|------|
| `node.exe` (index.js) | 8080 | HTTP API + WebSocket 服务器 |
| `postgres.exe` | 5432 | PostgreSQL 数据库 |

**查看进程（Windows）:**
```bash
# 查看 Node.js 进程
tasklist | findstr node

# 查看 PostgreSQL 进程
tasklist | findstr postgres

# 查看端口占用
netstat -ano | findstr 8080
```

**停止服务（Windows）:**
```bash
# 停止 Node.js 服务器
taskkill /F /IM node.exe

# 或使用进程 PID
taskkill /F /PID <进程 ID>
```

## 登录与注册

### 首次使用 - 注册账号

1. 启动客户端后，进入登录页面
2. 点击 **"注册账号"** 链接
3. 填写信息：
   - **用户名**: 唯一标识，用于登录
   - **密码**: 至少 6 位
   - **昵称**: 显示名称（可选）
4. 点击 **"注册"** 按钮
5. 注册成功后自动返回登录页

### 登录流程

1. 输入 **用户名** 和 **密码**
2. （可选）勾选 **"记住密码"**
3. （可选）勾选 **"自动登录"**
4. 点击 **"登录"** 按钮

**登录成功标志:**
- 窗口从登录页切换到主聊天界面
- 左上角显示当前用户名和头像
- 自动连接到 WebSocket 服务器
- 加载好友列表和消息历史

### 忘记密码

1. 点击登录页 **"忘记密码"** 链接
2. 输入用户名
3. 点击 **"获取验证码"**
   - 系统生成 8 位验证码（调试模式直接显示）
   - 有效期 15 分钟
4. 输入验证码和新密码
5. 点击 **"重置密码"**

## 常用功能

### 1. 发送消息

1. 在左侧会话列表选择联系人
2. 在底部输入框输入消息
3. 按 `Enter` 发送，或点击发送按钮

### 2. 添加好友

**方式一：搜索添加**
1. 点击顶部 **"+"** 按钮
2. 选择 **"添加联系人"**
3. 输入对方用户名或用户 ID
4. 点击 **"发送好友申请"**

**方式二：接收申请**
1. 收到好友申请通知
2. 点击 **"好友申请"** 查看
3. 选择 **"接受"** 或 **"拒绝"**

### 3. 查看好友列表

- 左侧边栏自动显示已添加的好友
- 在线好友显示绿色状态点
- 点击好友头像开始聊天

### 4. 修改密码（登录后）

1. 点击右上角用户名
2. 选择 **"设置"**
3. 点击 **"修改密码"**
4. 输入旧密码和新密码
5. 点击 **"确认修改"**

### 5. 退出登录

1. 点击右上角用户名
2. 选择 **"设置"**
3. 点击底部 **"退出登录"** 按钮

## 故障排查

### 后端服务问题

#### 1. 端口被占用

**错误:** `Error: listen EADDRINUSE: address already in use :::8080`

**解决:**
```bash
# 查找占用 8080 端口的进程
netstat -ano | findstr 8080

# 停止占用进程（假设 PID 为 12345）
taskkill /F /PID 12345

# 或修改 server/.env 中的 PORT
PORT=8081
```

#### 2. 数据库连接失败

**错误:** `Error: connect ECONNREFUSED 127.0.0.1:5432`

**解决:**
```bash
# 检查 PostgreSQL 服务
sc query postgresql-x64-14

# 启动服务
net start postgresql-x64-14

# 检查 .env 配置是否正确
# DB_HOST=localhost
# DB_PORT=5432
# DB_USER=postgres
# DB_PASSWORD=你的密码
```

#### 3. 缺少模块

**错误:** `Cannot find module 'xxx'`

**解决:**
```bash
cd E:\Project\QtProject\ChatApp\server
npm install
```

### 客户端问题

#### 1. 无法连接服务器

**现象:** 登录后一直显示 "连接中..."

**检查:**
1. 确认后端服务已启动（见 [启动后端服务](#启动后端服务)）
2. 检查防火墙是否阻止 8080 端口
3. 查看客户端日志：
   ```
   [NetworkClient] Initialized with Base URL: http://localhost:8080/api
   [WebSocketClient] Connecting to: ws://localhost:8080
   ```

#### 2. 登录失败

**错误:** "用户名或密码错误"

**检查:**
1. 确认用户名和密码正确
2. 检查后端日志是否有数据库错误
3. 尝试注册新账号测试

#### 3. 消息发送失败

**现象:** 消息显示在本地，但对方收不到

**检查:**
1. WebSocket 连接状态（查看底部状态栏）
2. 双方是否已建立好友关系
3. 后端日志是否有消息处理错误

### 日志查看

Node.js 服务器启动后，控制台实时输出日志：

## 快速启动脚本

### Windows 批处理（启动后端）

创建 `start_server.bat`:
```batch
@echo off
echo Starting ESChat Backend Server...
cd /d E:\Project\QtProject\ChatApp\server
node index.js
pause
```

### Windows 批处理（完整启动）

创建 `start_all.bat`:
```batch
@echo off
echo ========================================
echo   ESChat - 一键启动脚本
echo ========================================

echo [1/3] 检查 PostgreSQL 服务...
sc query postgresql-x64-14 | findstr "RUNNING"
if errorlevel 1 (
    echo PostgreSQL 未运行，正在启动...
    net start postgresql-x64-14
) else (
    echo PostgreSQL 已运行
)

echo [2/3] 启动后端服务器...
start "ESChat Server" cmd /k "cd /d E:\Project\QtProject\ChatApp\server && node index.js"

echo [3/3] 等待服务器启动...
timeout /t 3 /nobreak

echo 启动客户端...
start "" "E:\Project\QtProject\ChatApp\bin\ChatApp.exe"

echo ========================================
echo   启动完成！
echo ========================================
```
