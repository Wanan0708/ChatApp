# ESChat 运行与部署指南

本文档覆盖当前项目的本地联调、客户端启动方式和通用部署约定，替代旧的固定 IP 和单机面板文档。

如果需要部署到阿里云 ECS，可参考：docs/aliyun_bt_panel_guide.md。

## 本地运行

### 后端依赖

- Node.js 18+
- PostgreSQL 14+
- npm 8+

### 数据库初始化

```bash
createdb -U postgres chatapp
psql -U postgres -d chatapp -f server/init_db.sql
```

如需执行增量迁移，可运行：

```bat
server\migrations\run_migrations.bat
```

当前后端启动时只校验 schema，不再自动执行 ALTER TABLE 补字段；如果数据库缺少增量字段，服务会直接报错并提示先执行迁移。

### 后端配置

server/.env 最少应包含：

```env
PORT=8080
DB_HOST=localhost
DB_PORT=5432
DB_NAME=chatapp
DB_USER=postgres
DB_PASSWORD=123456
JWT_SECRET=replace_me
```

### 启动后端

```powershell
./scripts/start-local-server.ps1
```

### 启动客户端

本地联调建议统一使用：

```powershell
./scripts/start-local-client.ps1
```

这个脚本会强制注入以下环境变量：

- CHATAPP_BACKEND_HOST=127.0.0.1
- CHATAPP_BACKEND_PORT=8080
- CHATAPP_API_SCHEME=http
- CHATAPP_WS_SCHEME=ws

原因是发布版客户端在 Release 下默认会使用远端主机地址，直接双击 bin/ChatApp.exe 不适合本地联调。

## 从源码运行客户端

优先使用 Qt Creator 打开 ChatApp.pro。

如果使用命令行，请根据本机工具链选择对应构建命令，例如 qmake + mingw32-make 或 qmake + nmake。不要再假定 make 和 ./ChatApp 这组 Unix 命令在 Windows 下可直接使用。

## 部署约定

### 后端

- 入口文件：server/index.js
- 对外端口：8080
- 反向代理目标：127.0.0.1:8080
- 静态站点同样由 Node 服务提供

### 客户端

- Windows 发布目录通常以 bin/ChatApp.exe 为入口
- 如果部署到生产环境，应通过环境变量或打包阶段配置明确后端地址，而不是依赖默认常量

### 网站

- 首页：/
- 下载页：/download.html
- 关于页：/about.html
- 下载资源目录：web/downloads

## 排障要点

### 后端可访问但客户端仍连接失败

先确认客户端进程启动时是否带了本地覆盖环境变量。直接双击发布版 exe 很容易连到默认远端地址。

### 数据库脚本使用混乱

当前以 server/init_db.sql 作为基础建库脚本；database 目录下的内容保留作数据库侧说明，不再作为主启动文档入口。

### 迁移脚本执行失败

确认：

- psql 已加入 PATH
- CHATAPP_DB_HOST、CHATAPP_DB_PORT、CHATAPP_DB_NAME、CHATAPP_DB_USER、CHATAPP_DB_PASSWORD 设置正确
- schema_migrations 表可创建
